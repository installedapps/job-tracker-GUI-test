Job Tracker is a dark, local-first Rust desktop app for managing job applications,
notes, dates, pipeline status, and visual progress without sending your search
data anywhere.

toc::[]

== Highlights

[cols="1,2", frame=none, grid=rows]
|===
|Local-first storage |Applications are stored in SQLite on your machine.
|Focused workflow |Applications and Stats live in separate tabs.
|Pipeline tracking |Track Applied, Interviewing, Rejected, and Pending states.
|Calendar input |Add application dates with an inline calendar.
|Visual feedback |Review animated status bars, summary stats, and a heatmap.
|SVG export |Export a Sankey-style view of the job pipeline.
|===

== Quick Start

=== Prerequisites

* Rust stable toolchain
* A desktop environment supported by `iced`

=== Run

[source,bash]
----
cargo run
----

=== Test

[source,bash]
----
cargo test
cargo clippy --all-targets -- -D warnings
----

== Data Location

The app stores its SQLite database in the platform app data directory via the
`directories` crate. Sankey exports are written beside that data file as:

== Notes

The UI keeps animation work scoped to the Stats tab so the app does not redraw
unnecessarily while managing applications. Sankey export runs through an Iced
task so file writing does not block the update loop.
